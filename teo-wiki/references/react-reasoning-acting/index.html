ReAct: Synergizing Reasoning and Acting in Language Models

A foundational agent-pattern paper: interleave reasoning traces with actions that query tools, environments, or external knowledge.

Concept in one page

Core idea

Reasoning and acting should be interleaved

ReAct asks the model to alternate between internal reasoning steps and external actions. The model can think about what it knows, decide what to do next, call a tool or interact with an environment, observe the result, and then update its next reasoning step.

Korean quick note

생각만 하는 CoT에서, 행동하는 agent로 넘어가는 다리

Chain-of-Thought가 “문제 풀이 과정을 말로 풀어내는 방식”이라면, ReAct는 거기에 Action을 붙인다. 즉, 모델이 추론하다가 검색, 도구 호출, 환경 조작 같은 행동을 하고, 관찰 결과를 다시 추론에 반영한다.

Pattern

Thought → Action → Observation → Thought

The practical template is a loop: generate a thought, choose an action, receive an observation, then continue reasoning. This is close to how many modern tool-using agents are implemented.

Why this matters for LLM agents

  • Grounding: actions let the model gather missing information from tools or environments instead of relying only on parametric memory.
  • State tracking: reasoning traces help maintain a plan across multiple steps and recover from unexpected observations.
  • Agent design: the paper gives a simple conceptual blueprint for tool-using agents: reason, act, observe, revise.
  • Connection to Teo Wiki: this is directly relevant to how an agent can maintain a public/private research wiki, browse sources, update pages, and verify results.

Study questions

  1. When is it better to let the model act externally rather than reason longer internally?
  2. How should an agent decide which observation is trustworthy enough to update its plan?
  3. How does ReAct relate to current agent loops such as plan-act-observe, tool calling, and browser/terminal agents?
