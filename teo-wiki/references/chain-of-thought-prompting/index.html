Chain-of-Thought Prompting Elicits Reasoning in Large Language Models

A foundational prompting paper: give the model examples with intermediate reasoning steps so it learns to produce step-by-step reasoning for hard tasks.

Concept in one page

Core idea

Show the model the reasoning path, not only the answer

Chain-of-Thought prompting provides few-shot examples where each answer includes intermediate reasoning steps. Large models can imitate this format and solve tasks that require multi-step reasoning more reliably.

Scaling behavior

Reasoning ability appears more clearly in sufficiently large models

The paper emphasizes that CoT gains are strongest for large language models, suggesting that some reasoning behaviors become easier to elicit as model scale increases.

Why this matters for LLM agents

  • Reasoning interface: CoT made reasoning traces a first-class prompting object, influencing later agent frameworks.
  • Planning precursor: many agent loops rely on a model producing an explicit plan or rationale before actions.
  • Evaluation caveat: a generated chain of thought can improve performance, but it is not automatically a faithful explanation of the model’s internal computation.
  • Connection to ReAct: ReAct can be read as extending CoT by adding actions and observations between reasoning steps.

Study questions

  1. Which tasks benefit from explicit reasoning traces, and which tasks do not?
  2. When should an agent expose reasoning to users versus keep it as internal scratch work?
  3. How does CoT relate to later methods such as self-consistency, tool use, ReAct, and inference-time scaling?
